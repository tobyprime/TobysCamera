# 管理员地图画图库设计

## 目标

为拥有 `tobyscamera.admin` 权限的管理员提供一个游戏内箱子式图库，用于快捷查询、预览和管理已经保存的地图画。管理员可以按上传玩家搜索和筛选、翻页、切换时间排序、领取照片袋、删除照片，以及禁用或解除玩家的后续上传资格。

## 范围

- 新增 `/tobyscamera gallery` 管理员入口和箱子式 GUI。
- 支持按玩家名、玩家 UUID 前缀和照片 UUID 前缀搜索，支持按创建时间最新或最早排序与分页。
- 详情页按需预览一张地图画，显示完整拍摄 metadata，并可领取照片袋、确认删除照片、禁用或解除上传者。
- 在插件自己的 SQLite 数据库维护上传禁用名单；不依赖 LuckPerms、Vault 或其他权限插件。
- 不改变 Fabric 客户端协议、图片像素格式、已有地图物品或展示框中的物品。

不在范围内：中断已开始的上传会话。禁用只阻止该玩家下一次发起新的上传。

## 数据模型

### 照片归属名称

`photos` 表新增可空的 `owner_name` 列。新上传保存创建时玩家的名称，确保按名称筛选不依赖外部服务。

已有记录仍只有 `owner` UUID：查询展示时尝试从 Bukkit 的离线玩家缓存解析名称，不能解析时显示 UUID；历史记录仍可按 UUID 或照片 ID 搜索。不会回写或修改既有图片数据。

### 拍摄 metadata

新增以照片 ID 为主键的 `photo_metadata` 表，与照片记录在同一保存事务中写入。它保存上传时的拍摄者名称、世界、方块坐标、拍摄时间、照片标题、描述，以及三个公开开关（拍摄者、地址、拍摄时间）。`PhotoRecord` 读取时包含可空 metadata，`MapPhotoService` 以它作为重新领取照片袋的唯一 metadata 来源。

旧照片没有对应记录时 metadata 为缺失状态；详情页显示“旧照片未保存拍摄信息”，而不是伪造字段。新字段不会要求迁移或修改旧照片的媒体容器。

照片仓库新增分页查询接口：

- 可选搜索词匹配上传者名称、上传者 UUID 前缀或照片 UUID 前缀；
- 可按创建时间升序或降序排序；
- 接受页号和固定页面大小，只返回当前页及是否仍有下一页；
- 只读取照片元数据和地图瓦片索引，不在浏览时加载所有预览或瓦片像素。

### 上传禁用

新增 `upload_blocks` 表，保存被禁用的玩家 UUID、执行管理员 UUID 和禁用时间。仓库提供查询、添加和删除禁用记录的操作。

`UploadCoordinator` 在接受新的 `UploadStart` 前，除既有 `tobyscamera.upload` 权限校验外检查此表。被禁玩家收到上传拒绝；已经建立的上传会话保持原状，直到正常完成或过期。

### 删除照片

删除在仓库中完成：删除照片元数据、预览、地图瓦片索引和对应的媒体容器文件，并使相关缓存项失效。删除失败时不刷新图库，并向管理员报告错误。

已在玩家背包或物品展示框中的旧照片物品不会被扫描或销毁；其记录删除后无法再从存储读取渲染内容，保持现有缺失媒体的处理方式。地图 ID 不回收，符合 Bukkit 不提供安全回收 API 的现有限制。

## 管理员 GUI

管理员执行 `/tobyscamera gallery` 后打开六行箱子 GUI。

列表页顶部控制栏提供：上一页、下一页、按玩家筛选、清除筛选、最新优先/最早优先切换和关闭。列表条目只展示上传者、创建时间、尺寸与照片 ID 短前缀。点击条目打开详情页。

筛选通过管理员聊天输入提交；输入的关键词用于玩家名称、UUID 前缀和照片 ID 前缀检索。没有结果或页码失效时显示状态物品，但保留筛选和导航控制。

详情页包含一张不可取走的预览地图、上传者、时间、尺寸和完整照片 ID。预览复用已保存的预览像素与现有虚拟地图交付机制；一个会话最多维护当前详情照片的一张预览地图，避免列表页批量分配虚拟地图。

详情页还使用只读信息物品展示完整拍摄 metadata：拍摄者、拍摄世界和坐标、拍摄时间、照片标题、描述及三项公开开关。管理员始终能看到全部原始字段，公开开关只说明普通照片袋对玩家显示这些字段的策略。metadata 缺失的历史照片显示单独状态物品。

详情操作为：

- 领取照片袋：生成带 `tobyscamera:photo_copy` 标记、可正常展开、没有“底片”提示的照片袋；它和由其展开的地图均不可再次复制。按现有投递规则放入管理员背包，背包满时落在玩家当前位置；
- 删除照片：先进入确认页，第二次点击确认后才删除；
- 禁用上传者：写入上传禁用表；
- 解除上传者禁用：删除该禁用记录；
- 返回列表或关闭界面。

所有图库按钮、预览物品、拖拽和热键转移都由 GUI 会话监听器拦截，管理员无法取走界面物品。关闭时释放本次会话和临时预览状态。数据库和媒体文件操作放在异步任务中，库存改动、聊天输入后的 GUI 刷新及预览交付回到对应玩家线程。

## 权限与命令

`gallery` 子命令及全部 GUI 操作要求 `tobyscamera.admin`。现有 `reload`、`status` 和 `detail` 行为不变。`plugin.yml` 的命令用法和管理员权限说明更新为包含图库。

## 错误处理

- 无匹配、空页或无法解析的历史玩家名会通过 GUI 状态物品明确提示。
- 查询、预览、领取、禁用或删除失败时，保持当前照片数据不变，向管理员发送简短失败信息并在服务端记录警告。
- 删除仅在数据记录和媒体容器都成功处理后才刷新列表；失败不伪装为删除成功。
- 上传禁用只在下一个上传请求检查，不主动终止已有会话。

## 测试

- SQLite 仓库：名称迁移与保存、metadata 的保存与旧记录缺失兼容、分页排序、名称/UUID/照片 ID 筛选、禁用持久化和照片删除后的媒体清理。
- 上传：被禁用玩家的新上传会被拒绝，未禁用玩家与既有权限行为保持不变。
- GUI：管理员权限、分页、排序、筛选、完整 metadata 展示、详情导航、领取的非底片不可复制语义、删除确认、禁用切换和界面物品不可取走。
- 执行 Folia 测试及完整 Gradle 验证，确认 Fabric 客户端和既有照片处理没有回归。
