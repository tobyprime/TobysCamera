# Photo Bag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver each finished photo or video as one preview-map photo bag that can expand into a complete item-frame grid and be recovered or unpacked safely.

**Architecture:** `PhotoBag` custom-data fields identify immutable stored media and its 1×1 preview `MapView`. Server-only services manufacture bags, derive six-direction item-frame grid positions, place/recover all member maps atomically, and preserve the existing copying recipe. Fabric sends one explicit `OpenPhotoBag` packet only after a one-second hold, so Paper never has to infer a key release it cannot observe.

**Tech Stack:** Java 21, Fabric API payloads, Paper/Folia region schedulers, Bukkit item frames/events, `minecraft:custom_data`, JUnit 5.
